# Contributing to BestOfLinux

> BestOfLinux.org is a community-driven project that welcomes contributions from developers, package maintainers, and Linux enthusiasts. This guide will help you get started with contributing to the project.

## Ways to Contribute

### 💻 Code Contributions

| Type | Description |
|------|-------------|
| 🐛 Bug Fixes | Fix issues and improve stability |
| ✨ New Features | Add new functionality |
| 🎨 UI/UX Improvements | Enhance user experience |
| 📝 Documentation | Improve project documentation |
| 🧪 Tests | Add or improve test coverage |

### 📦 Package Data Contributions

| Type | Description |
|------|-------------|
| 📋 Package Information | Add or update package details |
| 🔄 Installation Commands | Provide installation instructions |
| 🏷️ Tags and Metadata | Add relevant package metadata |
| 📊 Usage Statistics | Share package usage data |
| 📝 Package Reviews | Write package reviews |

### 👥 Community Support

| Type | Description |
|------|-------------|
| 💬 User Support | Help other users |
| 📚 Documentation | Write tutorials and guides |
| 🔍 Testing | Test new features |
| 🌐 Translation | Help with translations |
| 📢 Promotion | Spread the word |

## Development Setup

### 🛠️ Prerequisites

| Requirement | Description |
|-------------|-------------|
| 💻 Node.js | Version 18 or higher |
| 📦 npm | Latest stable version |
| 🔧 Git | Version control system |
| 📝 Code Editor | VS Code recommended |

### 🚀 Getting Started

1. **Fork the Repository**
   ```bash
   # Fork on GitHub
   # Clone your fork
   git clone https://github.com/your-username/bestoflinux.git
   cd bestoflinux
   ```

2. **Setup Development Environment**
   ```bash
   # Install dependencies
   npm install
   
   # Start development server
   npm run dev
   ```

3. **Create Feature Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Pull Request Process

### 📋 PR Checklist

1. **🔍 Code Review**
   - Passes all tests
   - Follows style guide
   - Includes documentation
   - No merge conflicts

2. **📝 Documentation**
   - Updated README if needed
   - Added code comments
   - Updated API docs
   - Added changelog entry

3. **🧪 Testing**
   - Added new tests
   - Updated existing tests
   - Passes CI checks
   - Tested locally

### 🚀 PR Submission

1. **Create Pull Request**
   - Clear title and description
   - Link related issues
   - Add screenshots if needed
   - Request reviews

2. **Address Feedback**
   - Respond to comments
   - Make requested changes
   - Update PR as needed
   - Keep PR up to date
