# Distribution Support

> BestOfLinux.org supports multiple Linux distributions, allowing you to discover, compare, and install packages across different package management systems. This guide explains our distribution support and specific features for each supported system.

## Supported Distributions Overview

BestOfLinux currently provides official support for five major Linux distribution ecosystems:

| Distribution | Package Manager | Description |
|--------------|----------------|-------------|
| üèπ Arch Linux | pacman/AUR | Rolling release with access to the latest software versions |
| üî∑ Debian | apt | Known for stability, security, and extensive package selection |
| üü† Ubuntu | apt | User-friendly distribution based on Debian with regular releases |
| üîµ Fedora | dnf | Cutting-edge features with a focus on open-source innovations |
| üì¶ Flathub | flatpak | Cross-distribution application delivery platform |

Our goal is to create a unified interface for discovering packages across all these distributions, making it easier to find the right software regardless of your preferred Linux flavor.

## Distribution-Specific Features

### üèπ Arch Linux

  <div className="fd-accordion-item">
    <h3 className="fd-accordion-trigger">

      <span className="fd-accordion-icon">+</span>
    </h3>
    <div className="fd-accordion-content">
      <p>Arch Linux uses <strong>pacman</strong> as its native package manager and offers the <strong>Arch User Repository (AUR)</strong> for community-maintained packages.</p>
      
      <h4>Key Features:</h4>
      <ul>
        <li>Access to official repositories and AUR packages</li>
        <li>Binary packages from official repositories</li>
        <li>Build-from-source options via AUR</li>
        <li>AUR helpers (yay, paru) support</li>
      </ul>
    </div>
  </div>
  
  <div className="fd-accordion-item">
    <h3 className="fd-accordion-trigger">
      Installation Commands
      <span className="fd-accordion-icon">+</span>
    </h3>
    <div className="fd-accordion-content">
      <h4>For Official Repositories:</h4>
      
      ```bash
      # Install a package
      sudo pacman -S package_name
      
      # Update package database and upgrade all packages
      sudo pacman -Syu
      
      # Search for a package
      pacman -Ss search_term
      ```
      
      <h4>For AUR Packages:</h4>
      
      ```bash
      # Using yay (recommended)
      yay -S package_name
      
      # Manual installation
      git clone https://aur.archlinux.org/package_name.git
      cd package_name
      makepkg -si
      ```
    </div>
  </div>
  
  <div className="fd-accordion-item">
    <h3 className="fd-accordion-trigger">
      Special Considerations
      <span className="fd-accordion-icon">+</span>
    </h3>
    <div className="fd-accordion-content">
      <ul>
        <li><strong>Rolling Release:</strong> Packages are continually updated to the latest versions</li>
        <li><strong>AUR Caution:</strong> AUR packages should be reviewed before installation as they're community-maintained</li>
        <li><strong>Dependencies:</strong> Arch handles dependencies automatically but may require manual intervention for complex cases</li>
        <li><strong>Partial Updates:</strong> Not recommended as they can lead to system instability</li>
      </ul>
    </div>
  </div>
</div>

      <h4>Key Features:</h4>
      <ul>
        <li>Comprehensive package dependency resolution</li>

      ```bash
      # Install a package
      sudo apt install package_name
      
      # Update package lists and upgrade all packages
      sudo apt update && sudo apt upgrade
      
      # Search for a package
      apt search search_term
      
      # Install a local .deb file
      sudo apt install ./package_file.deb
      ```
    </div>

  <div className="fd-accordion-item">

      <span className="fd-accordion-icon">+</span>

        <li><strong>Release Cycle:</strong> Follows a fixed release model with long support cycles</li>
        <li><strong>Backports:</strong> Newer versions of packages can be found in the backports repository</li>
        <li><strong>Non-free Software:</strong> Some packages may require enabling the non-free repository</li>

## Installation Instructions

### General Guidelines

<div className="fd-steps [&_h3]:fd-step">

### 1. Verify System Compatibility
Check that your system meets the minimum requirements for the package.

- Consider disk space requirements
- Verify architecture compatibility (x86_64, ARM, etc.)
- Check kernel version requirements if applicable

### 2. Update Package Database
Always update your package database before installing new software.

- For Arch: `sudo pacman -Sy`
- For Debian/Ubuntu: `sudo apt update`
- For Fedora: `sudo dnf check-update`
- For Flatpak: `flatpak update`

### 3. Install the Package
Use the appropriate command for your distribution.

- Copy the command directly from the package detail page
- Review any additional flags or options included in the command
- Consider adding recommended dependencies if mentioned

### 4. Verify Installation
Confirm the package installed correctly.

- Check for error messages during installation
- Try launching the application
- Verify the expected version was installed

</div>

### Troubleshooting Tips

<div className="fd-accordion">
  <div className="fd-accordion-item">
    <h3 className="fd-accordion-trigger">
      Common Package Conflicts
      <span className="fd-accordion-icon">+</span>
    </h3>
    <div className="fd-accordion-content">
      <ul>
        <li><strong>Solution:</strong> Check for conflicting packages with similar functionality</li>
        <li><strong>For Arch:</strong> Use <code>pacman -Qo [file]</code> to find which package owns a conflicting file</li>
        <li><strong>For Debian/Ubuntu:</strong> Use <code>dpkg -S [file]</code> to identify package conflicts</li>
        <li><strong>For Fedora:</strong> Use <code>rpm -qf [file]</code> to find which package owns a file</li>
      </ul>
    </div>
  </div>
  
  <div className="fd-accordion-item">
    <h3 className="fd-accordion-trigger">
      Dependency Issues
      <span className="fd-accordion-icon">+</span>
    </h3>
    <div className="fd-accordion-content">
      <ul>
        <li><strong>Missing Dependencies:</strong> Ensure all required repositories are enabled</li>
        <li><strong>Version Mismatch:</strong> Check for minimum version requirements</li>
        <li><strong>For Arch:</strong> Try installing dependencies manually before the main package</li>
        <li><strong>For Debian/Ubuntu:</strong> Use <code>apt-cache policy [package]</code> to check available versions</li>
        <li><strong>For Fedora:</strong> Try <code>dnf --best --allowerasing install [package]</code> for complex dependency issues</li>
      </ul>
    </div>
  </div>
  
  <div className="fd-accordion-item">
    <h3 className="fd-accordion-trigger">
      Repository Problems
      <span className="fd-accordion-icon">+</span>
    </h3>
    <div className="fd-accordion-content">
      <ul>
        <li><strong>Package Not Found:</strong> Ensure you have the correct repositories enabled</li>
        <li><strong>For Arch AUR:</strong> Check if the package was moved or renamed</li>
        <li><strong>For Ubuntu PPAs:</strong> Verify PPA is compatible with your Ubuntu version</li>
        <li><strong>For Fedora:</strong> Check if the package is in RPM Fusion or COPR repositories</li>
        <li><strong>For Flatpak:</strong> Ensure Flathub is properly added as a remote</li>
      </ul>
    </div>
  </div>
  
  <div className="fd-accordion-item">
    <h3 className="fd-accordion-trigger">
      Installation Fails
      <span className="fd-accordion-icon">+</span>
    </h3>
    <div className="fd-accordion-content">
      <ul>
        <li><strong>Permission Errors:</strong> Ensure you're using <code>sudo</code> when required</li>
        <li><strong>Disk Space:</strong> Verify sufficient space is available</li>
        <li><strong>Internet Connection:</strong> Check your connection stability</li>
        <li><strong>Package Corruption:</strong> Clear package cache and try again</li>

### Data Freshness Considerations

- **Package Availability Delay:** There may be a delay between when a package is updated in the official repository and when it appears on BestOfLinux
- **Version Accuracy:** Version information is as accurate as the data provided by each distribution's API
- **Installation Commands:** Installation commands are generated based on the latest available data
- **Repository Changes:** Major repository reorganizations may cause temporary inaccuracies

### Inclusion Criteria

For a distribution to be added to BestOfLinux, it must meet the following criteria:

1. **Significant User Base:** The distribution must have a substantial and active user community
2. **Stable Package API:** A reliable method to access package data programmatically
3. **Regular Maintenance:** Active development and security updates
4. **Distinct Package Ecosystem:** Offers unique value not covered by existing distributions
5. **Documentation:** Comprehensive installation documentation available
